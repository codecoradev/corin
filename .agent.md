# Agent Context — Codecora Hub

## Project

Codecora Hub — desktop knowledge workstation powered by Uteke.
Local-first, offline-capable Tauri 2 + Svelte 5 + Rust + SQLite app for managing
memories, knowledge graphs, and rooms.

## Stack

| Layer | Tech |
|-------|------|
| Desktop shell | Tauri 2 |
| Frontend | Svelte 5 (runes: `$state`, `$derived`, `$effect`, `$props`) |
| Backend | Rust (edition 2024) |
| Storage | SQLite via rusqlite (bundled) |
| Graph | D3.js v7 (force-directed) |
| Markdown | CodeMirror 6 + marked + dompurify |
| Theme | Catppuccin Mocha (dark) |

## Architecture

```
Svelte 5 Frontend (src/)
  └── Tauri IPC (invoke) — src/lib/ts/ipc.ts
       └── Rust Commands — src-tauri/src/commands.rs
            └── SQLite (uteke.db)
                 └── memories + graph_edges tables
```

### Key files

- `src/App.svelte` — root component (welcome screen + main layout)
- `src/lib/stores/ui.svelte.ts` — UI state (sidebar, panels, active view)
- `src/lib/stores/memories.svelte.ts` — memory CRUD store
- `src/lib/ts/ipc.ts` — Tauri invoke wrappers
- `src/lib/ts/types.ts` — shared types
- `src-tauri/src/commands.rs` — all Tauri commands
- `src-tauri/src/lib.rs` — Tauri builder + handler registration
- `src-tauri/src/uteke_adapter.rs` — Phase 2 placeholder (uteke-core integration)

## Phase Plan

- **Phase 1 (v0.1, current):** MVP — memory CRUD, graph viz, search, rooms
  - Backend: direct SQLite + rusqlite
  - Search: naive `LIKE %query%` (FTS5 + semantic di Phase 2)
- **Phase 2 (v0.2):** Markdown editor, import/export, migrate to uteke-core library
- **Phase 3 (v0.3):** Auto-linking (cosine similarity), daily notes, backlinks
- **Phase 4 (v0.4):** AI assistant (opt-in), summarization

## Branch Conventions

- All PRs target `develop` (not `main`)
- Feature branches: `feat/<issue-number>-<description>`
- Fix branches: `fix/<description>`
- **One issue = one branch = one PR = wait CI green = merge = next**
- Pre-commit hook (cora review) must pass before each commit

## Code Review

- **Pre-commit hook:** `cora review --staged --format compact` runs automatically
- **CI:** `cora-review.yml` runs on every PR via `codecoradev/cora-review-action@v1`
- Config: `.cora.yaml` (provider, focus, rules, ignore patterns)
- Required CI secrets: `CORA_API_KEY`, `CORA_BASE_URL`, `CORA_MODEL`

## Key Rules for Agents

- **Svelte 5 runes** (`$state`, `$derived`, `$effect`) are correct — do NOT use Svelte 4 `$:` syntax
- **All SQL** must use parameterized queries (`rusqlite::params![]` or `?N` placeholders)
- **Tauri commands** are async fns with `#[tauri::command]`, state via `tauri::State<Arc<Mutex<AppState>>>`
- **IPC types** must stay in sync between `src/lib/ts/types.ts` and `src-tauri/src/commands.rs`
- **CommandError** serializes to string for frontend — add new variants carefully

## CI Required Checks
